# ADR 0004 — 用独立的 openarm_labs_hub 仓库沉淀跨项目经验

- 日期：2026-06-17
- 状态：已采纳
- 领域：知识管理

## 背景
根目录下有 ~12 个仓库，多数是 enactic 上游克隆（只读），自有工作分散在 `openarm_mp_labs` / `GraspGenX` 两个 fork。经验散落在对话和单个项目的 to-do.md 里，易丢失。

## 问题
- 经验不能写进上游克隆（重拉即丢、推不上去）。
- 也不该塞进某个具体子项目（会让它失焦，跨领域知识找不到）。

## 决策
新建独立仓库 `openarm_labs_hub` 作为**知识中枢**：只放文档（仓库地图、gotchas、ADR、实验记录、playbooks、环境知识），不放实现代码。各子项目深度文档仍留各自 fork，中枢做索引+横切知识。

## 备选方案
- **复用 openarm_mp_labs**：会变成大杂烩，否决。
- **让根目录变成 git 仓库**：会与各子仓 .git 嵌套混乱，否决。

## 影响
新增维护约定（见 hub README）：碰坑记 gotchas、做判断写 ADR、成规模探索写实验记录、跑通流程写 playbook。
